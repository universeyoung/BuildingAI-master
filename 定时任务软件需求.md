## 1. 概述与目标

### 1.1 模块定义

定时任务模块允许用户创建、配置并调度周期性 AI 对话任务。系统在指定时间自动触发与智能体的对话（对话能力等价于"新对话"模块）。每次执行产生独立的定时任务记录，与手动对话记录区分存储和展示。

### 1.2 核心能力矩阵

| 能力维度 | 要求 | 参考模块 |
|----------|------|----------|
| 任务 CRUD | 列表、新建、编辑、启用/禁用、删除、手动运行 | — |
| 调度配置 | Cron 表达式 / 图形化选择器（如"每天 08:00"） | ACCIO WORK |
| 智能体选择 | 选择一个已创建的智能体 | 智能体模块 |
| 会话管理 | 新建会话 / 选择历史会话继续 | 新对话模块 |
| 提示词 | 自定义提示词模板 | 新对话模块 |
| 高级设置 | 模型选择、MCP 工具、文件上传、思考模式、应用选择 | 新对话顶部模型选择栏 |
| 对话执行 | 完整对话能力（流式、工具调用、推理、附件） | 新对话模块 |
| 执行记录 | 独立左侧栏「定时任务记录」，每次运行新增/追加记录 | 新对话历史记录 |

### 1.3 前置约束

- 用户已登录（通过 `AuthGuard`）
- 前端统一使用 `@buildingai/web/*` 封装
- 后端统一使用 `BaseService<T>` / `BaseController` / `@WebController`
- 数据库实体进入 `public` schema
- 对话能力复用现有 `@buildingai/ai-sdk` 和对话服务

export function cronToHumanReadable(expr: string, locale = "zh-CN"): string {
  // "0 8 * * *" → "每天 08:00"
  // "0 9 * * 1" → "每周一 09:00"
  // "0 */3 * * *" → "每3小时执行"

export function calculateNextRun(cronExpression: string, fromDate?: Date): Date {
  const interval = parseExpression(cronExpression, {
    currentDate: fromDate || new Date(),
  });
  return interval.next().toDate();
}

// 在执行对话的事务中
await this.taskRunRepo.manager.transaction(async (entityManager) => {
  // 1. 创建/更新 run 记录
  await entityManager.save(ScheduledTaskRun, { ...runUpdates });

  // 2. 扣费
  await this.appBillingService.deductUserPower(
    {
      userId: task.userId,
      amount: actualCost,
      accountType: ACCOUNT_LOG_TYPE.CHAT_DEC,

      associationNo: run.id,
    },
    entityManager,
  );

  // 3. 更新 task 统计
  await entityManager.increment(
    ScheduledTask,
    { id: task.id },
    "totalRunCount",
    1,
  );
});

## 13. 风险与注意事项

| # | 风险/注意 | 缓解措施 |
|---|----------|----------|
| R-01 | Cron 表达式用户编写困难 | 提供图形化选择器 + 实时预览 |
| R-02 | 大量任务同时触发造成服务压力 | 限制每用户最多 N 个启用任务；错峰执行 |
| R-03 | 对话执行中服务重启 | 使用 QueueService 持久化待执行任务 |
| R-04 | 与现有对话模块耦合 | 复用 ChatService 接口，不直接操作底层 |
| R-05 | 继续会话模式下原会话被删除 | 校验会话存在性，不存在时自动切换为新建模式 |
| R-06 | 会话上下文无限增长 | 限制继续会话的最大消息轮数（可配置） |
