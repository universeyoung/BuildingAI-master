# 辰启-自动采集 Skill v24.2（精简字段版）

## 核心定位

6步闭环：动态读取 → 精简字段采集+扁平化 → 字段补全 → 写入表格 → 完整性校验 → AI估算回填。

**采集字段（共10个）**：标题、描述简介、价格、运费、重量、长度、宽度、高度、材质、各规格主图URL

| 步骤 | 名称 | 关键输出 |
|------|------|---------|
| 第一步 | 动态读取待采集任务 | baseId、tableId、fieldId映射、目标记录+商品URL |
| 第二步 | 精简字段深度采集+SKU扁平化 | sku_list（每规格独立行，含专属高清主图） |
| 第三步 | 缺失字段自动新增 | 更新后的fieldId映射，确保表格结构完整 |
| 第四步 | 写入多维表格 | all_record_ids（原始记录+新增规格行） |
| 第五步 | 完整性逻辑校验 | critical_empty（重量/尺寸缺失记录列表） |
| 第六步 | AI智能估算回填 | 所有记录重量/尺寸无空值，追加[AI估算]标注 |

## 固定参数（可配置）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 目标表格名 | 商品信息采集 | 首次运行时按此名称查找 baseId |
| 数据表名 | 商品采集 | 表格内的具体 Sheet 名称 |
| 状态字段过滤值 | 待采集 | Python客户端过滤条件 |
| baseId | （空） | 若用户明确提供则直接使用，跳过名称查找 |

## 第一步：动态读取待采集任务

### 1.1 获取 baseId

**有配置**：用户在对话中明确提供 baseId，直接使用。

**无配置（首次/多账号）**：
```bash
dws aitable base search --keyword "商品信息采集" --format json
```
返回多个结果取第一个；无结果则尝试关键词「商品」再次搜索；仍无结果则终止并提示「未找到目标表格，请确认表格名称」。提示用户记录 baseId 以加速后续运行。

### 1.2 获取 tableId 与 fieldId 映射

⚠️ **正确命令**：`dws aitable table get`（非 `table list`）

```bash
dws aitable table get --base-id <baseId> --format json
```

从返回的 `tables` 数组中匹配 `tableName == "商品采集"` 提取 `tableId`；同时提取 `fields` 数组构建 fieldId 字典：

```python
field_id_map = {f['fieldName']: f['fieldId'] for f in table['fields']}
# 示例结果：{'标题':'TbcSlAV','状态':'MOgjNsj','商品链接':'pGqOzzN',...}
```

### 1.3 读取待采集记录（支持翻页）

```python
all_records, cursor, pending = [], None, []
status_fid = field_id_map.get('状态','')
while True:
    cmd = ['dws','aitable','record','query','--base-id',baseId,
           '--table-id',tableId,'--limit','100','--format','json']
    if cursor: cmd += ['--cursor', cursor]
    r = subprocess.run(cmd, capture_output=True, text=True, encoding='utf-8')
    d = json.loads(r.stdout)
    recs = d.get('data',{}).get('records',[])
    # ⚡ 优化：翻页提前终止——找到第一条「待采集」记录即停止
    for rec in recs:
        if rec['cells'].get(status_fid,{}).get('name') == '待采集':
            pending = [rec]
            break
    if pending: break
    all_records.extend(recs)
    cursor = d.get('data',{}).get('nextCursor')
    if not cursor or not recs: break

# Python客户端过滤（dws --filters 对singleSelect不准确）
```

若 `pending` 为空，输出「无待采集任务，请在表格中添加商品链接并将状态设为「待采集」」，正常终止。

取第一条，立即更新状态为「采集中」：
```python
subprocess.run(['dws','aitable','record','update','--base-id',baseId,
    '--table-id',tableId,'--records',
    json.dumps([{'recordId':primary_id,'cells':{status_fid:'采集中'}}]),'--yes'])
```

提取商品链接（url字段格式为 `{"text":"...","link":"https://..."}`）：

**重量单位标准化**（统一为整数 g）：
```python
def parse_weight_to_g(raw_val):
    if raw_val is None: return None
    try:
        v = float(str(raw_val).replace('g','').replace('G','').replace('kg','').strip())
        if 'kg' in str(raw_val).lower() or (v < 10 and v > 0):
            v = v * 1000
        return int(round(v)) if v > 0 else None
    except: return None
```

**尺寸单位标准化**（统一为 float cm）：
```python
def parse_dim_to_cm(raw_val):
    if raw_val is None: return None
    try:
        s = str(raw_val).lower().replace('cm','').replace('mm','').strip()
        v = float(s)
        if 'mm' in str(raw_val).lower(): v = round(v/10, 1)
        return round(v, 1) if 0 < v < 500 else None
    except: return None
```

## 第四步：写入多维表格

### 4.1 数据清洗规则

- **标题**：截断至200字符
- **商品主图URL**：确保以 http 开头，否则置为空字符串
- **重量/长度/宽度/高度**：使用转换函数；转换失败或值为0 → 置为 None（触发AI估算）
- **价格**：转为 float，失败则置为 None
- **运费**：保留文本原值，如「¥5.00起」
- **材质**：截断至100字符
- **描述简介**：截断至500字符

### 4.2 构建写入 cells（精简字段版）

```python
def build_cells(sku, field_id_map, product_url, status):
    img = sku.get('main_image','')
    # 价格：从原始文本提取第一个数字，转为float；失败则置None
    price_raw = sku.get('price', '')
    try:
        price_val = float(re.search(r'[\d.]+', str(price_raw)).group()) if price_raw else None
    except (AttributeError, ValueError):
        price_val = None
    cells = {
        field_id_map['标题']:        sku['title'][:200],
        field_id_map['商品链接']:    {'text':'商品链接','link':product_url},
        field_id_map['价格']:        price_val,
        field_id_map['运费']:        sku.get('freight',''),
        field_id_map['描述简介']:    sku.get('description','')[:500],
        field_id_map['重量']:        sku.get('weight'),
        field_id_map['长度']:        sku.get('length'),
        field_id_map['宽度']:        sku.get('width'),
        field_id_map['高度']:        sku.get('height'),
        field_id_map['材质']:        sku.get('material','')[:100],
        field_id_map['商品主图URL']: {'text':'主图','link':img} if img else None,
        field_id_map['状态']:        status
    }
    return {k: v for k, v in cells.items() if v is not None}
```

## 第五步：完整性逻辑校验

```python
def is_missing(val):
    if val is None or val == '': return True
    try: return float(val) == 0
    except: return False

critical_empty = []
for sku, record_id in zip(sku_list, all_record_ids):
    w_miss = is_missing(sku.get('weight'))
    s_miss = (is_missing(sku.get('length')) or
              is_missing(sku.get('width'))  or
              is_missing(sku.get('height')))
    if w_miss or s_miss:
        critical_empty.append({
            'recordId':    record_id,
            'title':       sku.get('title', ''),
            'main_image':  sku.get('main_image', ''),
            'description': sku.get('description', ''),
            'missing':     (['重量'] if w_miss else []) + (['尺寸'] if s_miss else [])
        })
```

若 `critical_empty` 为空，输出「完整性校验通过，无需AI估算」，跳至收尾。否则进入第六步。

## 第六步：AI 智能估算与回填

**触发条件**：仅当第五步检测到 `critical_empty` 非空时执行。

### 6.1 逐条估算（类目规则优先 + 视觉分析兜底）

```python
category_rules = load_category_rules()
vision_cache = {}

def match_category_rules(title):
    if not title: return None
    best_match, best_score = None, 0
    for rule in category_rules:
        score = sum(1 for kw in rule['keywords'] if kw in title.lower())
        if score > best_score:
            best_score = score
            best_match = rule
    if best_match and best_score >= best_match.get('min_matches', 2):
        return {'defaults': best_match['defaults'],
                'confidence': min(best_score / len(best_match['keywords']), 1.0)}
    return None

for record in critical_empty:
    title = record.get('title', '')
    img_url = record.get('main_image', '')
    estimated = None

    category_match = match_category_rules(title)
    if category_match and category_match['confidence'] > 0.7:
        estimated = category_match['defaults']
    else:
        if img_url:
            if img_url in vision_cache:
                estimated = vision_cache[img_url]
            else:
                estimated = understand_media(
                    media_path=img_url,
                    question="推断商品重量(克)和尺寸(长宽高厘米)。返回JSON:{\"weight_g\":数字,\"length_cm\":数字,\"width_cm\":数字,\"height_cm\":数字}"
                )
                vision_cache[img_url] = estimated
        else:
            estimated = {'weight_g': 500, 'length_cm': 20, 'width_cm': 15, 'height_cm': 10}

    if estimated:
        estimated_weight = int(estimated.get('weight_g', 0))
        estimated_length = round(float(estimated.get('length_cm', 0)), 1)
        estimated_width  = round(float(estimated.get('width_cm', 0)), 1)
        estimated_height = round(float(estimated.get('height_cm', 0)), 1)
        if estimated_weight < 1 and estimated_weight > 0:
            estimated_weight *= 1000
        if estimated_weight <= 0 or estimated_weight > 100000:
            estimated_weight = category_match['defaults']['weight_g'] if category_match else 500
```

### 6.2 回填数据

```python
    update_record = {'recordId': record['recordId'], 'cells': {}}
    if '重量' in record['missing']:
        update_record['cells'][field_id_map['重量']] = estimated_weight
    if '尺寸' in record['missing']:
        update_record['cells'][field_id_map['长度']] = estimated_length
        update_record['cells'][field_id_map['宽度']] = estimated_width
        update_record['cells'][field_id_map['高度']] = estimated_height

    # AI估算标注追加到描述简介
    ai_note = '[AI估算]重量/尺寸为系统推算，建议人工核实'
    current_desc = record.get('description', '')
    if ai_note not in current_desc:
        update_record['cells'][field_id_map['描述简介']] = (current_desc+'\n'+ai_note).strip()

    subprocess.run(['dws','aitable','record','update','--base-id',baseId,
        '--table-id',tableId,'--records',json.dumps([update_record]),'--yes'])
```

## 1688 专项规则

- **主图选择器优先级**：`img.ant-image-img.preview-img` > `img.detail-gallery-img img` > `div.gallery-view-container img` > `img[data-spm="mainImage"]`
- **URL 四重清理**：去 `_sum.jpg` → 去 `_.webp` → 去 OSS 参数 → 补全 https 协议
- **懒加载处理**：属性区需滚动至1300px并使用动态等待检测元素可见
- **表格型商品检测**：存在 `#productPackInfo` 表格时，采用混合模式——从表格提取物理属性，同时遍历颜色选项采集专属高清主图。
- **Shadow DOM 限制**：若所有选择器均失败，尝试通过 shadowRoot 穿透查找
- **运费提取备选**：若 2.2 节 evaluate 未获取到运费，在属性区提取时查找「运费」/「快递费」关键词

## Token 优化策略（v23.0）

1. **精简采集字段**：仅采集10个目标字段，大幅减少写入数据量
2. **轻量 evaluate 脚本**：变量名缩短，文本限制200字符
3. **批量 API 调用**：字段查询、记录写入均使用批量接口
4. **缓存 fieldId 映射**：仅在新增字段后重新获取
5. **条件触发 AI 估算**：仅当重量/尺寸确实缺失时才估算
6. **AI估算策略反转**：类目规则优先，视觉分析兜底
7. **动态等待**：检测关键元素可见性，节省30-50%等待时间
8. **截图按需保存**：仅在关键失败节点保存

## 异常处理

| 异常场景 | 处理策略 |
|---------|---------|
| 无待采集记录 | 输出提示，正常终止 |
| 页面加载超时 | 截图保存至 tmp，报告并终止 |
| 检测到需要登录 | 截图提示用户手动登录，等待确认后继续 |
| 规格区未找到 | 尝试3种备选选择器，均失败则终止 |
| 主图采集连续失败 | 连续3个无效 URL，截图报告并终止 |
| 运费未提取到 | 写入空字符串，不终止流程 |
| 材质未提取到 | 写入空字符串，不终止流程 |
| SKU 数量超限 | 超过150条截断并记录警告，继续执行 |
| 字段新增失败 | 重试1次，仍失败则记录错误日志，跳过继续 |
| 写入记录失败 | 重试1次，仍失败则记录错误日志，继续处理 |
| AI 估算失败 | 降级至类目规则，再失败则使用全局默认值 |

## 验收标准

1. **精简字段采集**：稳定采集10个目标字段：标题、描述简介、价格、运费、重量、长度、宽度、高度、材质、各规格主图URL
2. **多规格主图准确**：每个规格对应专属高清主图，非缩略图
3. **表格写入正确**：数据一一对应，图片以 URL 字符串存储，状态流转正常
4. **材质字段采集**：从属性区正确提取材质，无法获取时写入空字符串
5. **运费字段采集**：从价格区域或属性区正确提取运费文本
6. **Token 消耗可控**：单次采集无冗余 API 调用，批量写入代替逐条写入
7. **零值判空准确**：重量或尺寸为0时正确触发 AI 估算
8. **登录识别可靠**：准确识别需要登录的场景并引导用户介入
