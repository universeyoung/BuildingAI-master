# cron 提示词写法范例

> Agent 用 LLM 把自然语言解析成 5 个结构化字段：关键词 / 数量 / 价格区间 / 品类排除 / 店铺链接

## 完整示例（5 维度全填）

```
夏季女装连衣裙，选 5 条，采购价 30-100，不要电子产品和易碎品
```

解析为：
```json
{
  "关键词": "夏季女装连衣裙",
  "数量": 5,
  "价格区间": [30, 100],
  "品类排除": ["电子产品", "易碎品"],
  "店铺链接": null
}
```

## 关键词模式（最常用）

```
家居收纳盒
家居收纳盒，选 3 条
家居收纳盒，¥50 以下
夏季拖鞋，选 5 条，月销 ≥100
```

## 店铺模式（你已锁定店铺）

```
https://shop123.1688.com/ 选 3 条
https://shop123.1688.com/ 选夏季款，采购价 30-100
```

> 店铺 + 关键词同时给 = 在该店铺内搜关键词（方案 ①）

## 兜底模式（让 Agent 自己决定）

```
开始选品
选品
```

> 触发 4 路市场调研 → 输出关键词池 → 随机抽 1 个

## 价格区间写法

支持的格式（全部正确解析）：

```
采购价 30-100        # 30 到 100
采购价 30~100
采购价 ¥30-100
¥30 以下             # 0-30
50 以上              # 50-150（上限取默认值）
```

## 品类排除写法

```
不要电子产品                # 排除 1 个
不要电子产品和易碎品          # 排除 2 个
不要电子、易碎品、食品        # 排除 3 个（顿号也支持）
排除 化妆品                  # "排除"也认
```

## 数量写法

```
选 3 条
选3个
3 条
共 3 条
```

## 不被接受的写法（会进 SOURCING_NO_KEYWORD）

```
xxx                      # 没具体含义
随便选选                  # 模糊
1688.com                 # 不是具体店铺链接
```

## cron 任务配置示例

```bash
# 每天 9:00-21:00 之间，每 20 分钟选 3 条家居用品
*/20 9-21 * * * "家居收纳盒，选 3 条"

# 每天 10:00 跑一次兜底调研
0 10 * * * "开始选品"

# 工作日 14:00 选服装
0 14 * * 1-5 "夏季女装，选 5 条，采购价 50-150"
```
